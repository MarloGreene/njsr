How does this work?

How is crack time calculated?

We calculate the total number of possible passwords by raising the character set size to the power of your password length.

  • Lowercase letters (a-z): 26 characters
  • Uppercase letters (A-Z): +26 characters
  • Numbers (0-9): +10 characters
  • Symbols (!@#$...): +32 characters

For example, an 8-character password using all types has 94^8 = 6 quadrillion combinations.

We then divide by 10 billion guesses per second (a realistic estimate for modern GPU-based cracking) to get the time estimate.

Is my password being stored or transmitted?

No. Everything happens right here in your browser using JavaScript. Your password never leaves your device.

Don't take our word for it — you can verify this yourself:

  • Open your browser's Developer Tools (F12)
  • Go to the Network tab
  • Type a password and watch — no requests are made
  • Or view the source code directly

This page works completely offline after loading.

Tips for strong passwords
  • Length matters most — 16+ characters is ideal
  • Mix character types — uppercase, lowercase, numbers, symbols
  • Avoid common patterns — "Password123!" is easily guessed
  • Use a passphrase — "correct horse battery staple" is strong and memorable
  • Never reuse passwords — use a password manager
  • Enable 2FA — adds protection even if password is compromised
Limitations of this tool

This calculator assumes a brute-force attack against a random password. Real-world cracking is often faster because:

  • Attackers use dictionary attacks with common passwords
  • They apply rules (e.g., replace 'a' with '@')
  • Leaked password databases help predict patterns
  • Weak hashing algorithms make cracking easier

Use this as a general guide, not a guarantee.
